We don't know them that much

Rest Specification was written 15 years ago

We don't respect half what the specification proposed

Specification author published a paper in 2008 (7 years ago) http://roy.gbiv.com/untangled/2008/rest-apis-must-be-hypertext-driven

Richardson Maturity Model

The Rest API Maturity chart

The Problems

  • Client apps have to hardcode the URL
  • Very HARD coupling with the apps that uses the API
  • Force to write documentation when building
  • RTFM when consuming
  • Very hard to discover by itself
  • Makes nearly impossible refactoring of the API itself

The Solution:

Hypermedia

  • Use Semantics instead of URI

    Speak like a human!

  • Add metadata to data
    • links
    • actions
    • embended objects
    • remote objects

Pluses

  • Consume instead of constructing
  • Detect API change, new links and react to novelty
  • In the end API can maintain the flow and Client only has to react to it
  • Finally URL can change without breaking clients!!!

Downs

  • Very new
  • Complicated to see benefits
  • Lake of tools
  • Great Oppertunities! Let's build the new hibernate of hateoas
